## 沟通机制

### 每日站会
- **时间:** 每天 09:30
- **形式:** 线上会议（腾讯会议）
- **内容:** 昨日进展、今日计划、阻塞问题

### 周例会
- **时间:** 每周五 14:00
- **形式:** 线下会议室 A
- **内容:** 本周总结、下周计划、风险评审

### 代码评审
- **平台:** GitHub Pull Request
- **要求:** 每个 PR 至少 1 人 review
- **流程:** 提交 PR → 自动 CI → 人工 review → 合并

## 开发规范

### 代码规范
- **后端:** Alibaba Java Coding Guidelines
- **前端:** ESLint + Prettier
- **提交信息:** Conventional Commits

### 分支策略
```
main            # 生产环境
release/*       # 发布分支
feat/*          # 功能分支
hotfix/*        # 热修复分支
```

### 测试要求
- **单元测试:** 覆盖率 > 80%
- **集成测试:** 关键路径覆盖
- **性能测试:** 核心接口压测
