# Datart 企业版改造方案文档

本目录包含 Datart 数据分析平台的企业级改造方案文档，旨在将 Datart 改造为适合大型集团企业使用的数据分析平台。

## 🎯 改造目标

将 Datart 改造为具备以下企业级特性的数据分析平台：

1. **多源连接** - 支持跨数据源查询、数据联邦、连接池管理
2. **集团化组织架构** - 多层级组织模型、权限继承、数据范围控制
3. **字段级权限** - 细粒度字段权限、动态脱敏、行级权限
4. **灵活图表控件** - 新增图表类型、图表联动、自定义图表 SDK
5. **耦合式集成** - 统一认证、前端 SDK、REST API 增强

## 🔐 核心特性

### 多源连接
- 跨数据源 JOIN 查询
- 数据联邦视图
- 企业级连接池管理（健康检查、故障转移、读写分离）

### 集团化组织架构
- 多层级组织模型（集团 - 事业部 - 部门 - 团队）
- 权限自动继承
- 数据范围控制

### 字段级权限
- 字段可见性控制（可见/脱敏/隐藏）
- 动态脱敏引擎（手机号、身份证、邮箱等）
- SQL 改写注入权限过滤

### 图表增强
- 新增桑基图、热力图、关系图、漏斗图、甘特图
- 图表联动机制（跨仪表板联动）
- 自定义图表 SDK

### 集成能力
- 统一认证（OAuth2、LDAP、JWT、CAS/SAML）
- 前端 SDK（@datart/sdk）
- 增强的 REST API 和 GraphQL API

## 📖 使用指南

### 开发环境搭建

```bash
# 克隆仓库
git clone https://github.com/your-org/datart.git
cd datart

# 切换到企业版分支
git checkout feat/enterprise-edition

# 启动开发环境
docker-compose -f docker-compose.dev.yml up -d
```

### 导入组织数据

```bash
# 使用批量导入 API
curl -X POST https://datart.example.com/api/org/import \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d @organizations.json
```

### 配置字段权限

通过管理界面配置数据集的字段级权限：
1. 进入数据集管理
2. 选择目标数据集
3. 点击"字段权限"
4. 为不同角色配置字段可见性和脱敏规则
